# LoreNest · 世界观设定管理工具

> 为小说作者、桌游 DM、游戏设计师打造的**本地优先**设定库。  
> 所有数据存储在本机，无需联网，无订阅费用。

---

## 它解决什么问题

创作一个庞大的世界观时，设定散落在 Word 文档、便签、脑图和记忆里。  
LoreNest 把这些碎片集中到一个地方：每个角色、地点、势力、事件都是一个**节点**，节点之间可以建立关系，形成一张可视化的知识网络。

---

## 核心功能一览

### 节点系统
每条设定信息都是一个「节点」，内置 9 种类型，用颜色区分：

| 类型 | 用途 |
|------|------|
| 角色 | 人物、生物、NPC |
| 地点 | 城市、地标、地图区域 |
| 势力 | 组织、国家、派系 |
| 事件 | 历史事件、战役、转折点 |
| 物品 | 道具、魔法物品、遗物 |
| 术语 | 世界观专有名词、魔法体系、规则 |
| 章节 | 故事章节结构 |
| 场景 | 具体场景片段 |
| 文档 | 自由格式的说明文档 |

每个节点有**4 种状态**管理写作进度：草稿 / 正典 / 归档 / 废弃。

---

### Markdown 编辑器
- 全功能 Markdown 编辑，支持编辑 / 预览 / 左右分栏三种视图
- 输入 `[[` 触发节点名自动补全（WikiLink），快速在文章内链接其他设定
- 自动保存，无需手动按 Ctrl+S

### 自定义字段模板
对每种节点类型定义专属字段，例如：
- 角色：年龄、种族、阵营、技能等级…
- 地点：气候、人口、统治者…
- 事件：时间线位置、参与势力、后果…

支持文本、数字、开关、日期、下拉单选、多选、标签、节点引用等 10 种字段类型。

### 关系网络
- 为任意两个节点建立有向关系（如「角色 A → 效忠 → 势力 B」）
- **关系图谱**页面：基于 React Flow 的交互式节点图，支持拖拽布局、缩放、小地图

### 时间线视图
将含有时间信息的节点（事件、场景、章节等）排列在横向时间轴上，直观展示世界历史顺序。

### 数据库视图
按类型展示节点的表格视图，支持搜索和按字段排序，适合管理大量同类条目。

### 仓库统计
- 节点类型分布条形图
- 各状态节点数量与占比
- 最近修改记录

---

## 数据安全

- **全本地存储**：使用 SQLite 数据库（WASM 版），文件保存在用户电脑上
- **快照备份**：一键对整个仓库创建时间点快照，可随时回滚到任意历史状态
- **Markdown 导出**：将所有节点导出为带 YAML frontmatter 的 `.md` 文件，可用任何文本工具打开

---

## 界面风格

「Deep Spire」主题：深靛蓝黑底色 + 暖紫白文字 + 金 / 青 / 紫三色强调系统。  
使用 Rajdhani（标题）和 JetBrains Mono（代码/数据）本地字体，离线环境下字体表现一致。

| 层级 | 技术 |
|------|------|
